# Markdown Parser 문서

이 문서는 Markdown Parser 프로젝트의 내부 구조, 타입 정의, 그리고 API 사용법에 대한 상세한 정보를 제공합니다. 본 프로젝트는 CommonMark 사양을 준수하며 MDAST 호환 출력을 생성하는 TypeScript 기반의 마크다운 파서입니다.

## 문서 목차

1. [아키텍처 개요 (Architecture)](./architecture.md)
   - 전체적인 파싱 흐름과 데이터 구조에 대해 설명합니다.
2. [타입 정의 (Types)](./types.md)
   - MDAST 기반의 노드 타입과 계층 구조를 상세히 다룹니다.
3. [토크나이저 (Tokenizer)](./tokenizer.md)
   - 라인 기반 토큰화 방식과 위치 추적 메커니즘을 설명합니다.
4. [블록 파서 (Block Parser)](./block-parser.md)
   - 블록 레벨 요소의 파싱 알고리즘과 재귀적 하강 파싱 방식을 설명합니다.
5. [인라인 파서 (Inline Parser)](./inline-parser.md)
   - 인라인 요소 파싱을 위한 델리미터 스택 알고리즘을 상세히 다룹니다.
6. [API 레퍼런스 (API)](./api.md)
   - 프로젝트에서 제공하는 주요 함수와 타입의 사용법을 안내합니다.

## 빠른 탐색 가이드

- **구현 구조가 궁금하신가요?** [아키텍처 개요](./architecture.md)를 먼저 읽어보세요. 파서가 어떻게 문자열을 토큰으로 나누고, 블록 구조를 형성하며, 최종적으로 인라인 요소를 처리하는지 전체적인 그림을 그릴 수 있습니다.
- **어떤 노드 타입이 있는지 알고 싶으신가요?** [타입 정의](./types.md) 문서에 모든 정의가 포함되어 있습니다. MDAST 사양을 따르는 각 노드의 속성과 인터페이스를 확인할 수 있습니다.
- **파서를 직접 사용하고 싶으신가요?** [API 레퍼런스](./api.md)에서 `parse` 함수의 사용법을 확인하세요. 설치 방법부터 기본적인 사용 예제, 그리고 고급 활용 방법까지 상세히 안내합니다.
- **내부 파싱 로직을 깊게 이해하고 싶으신가요?** [토크나이저](./tokenizer.md), [블록 파서](./block-parser.md), [인라인 파서](./inline-parser.md) 순으로 읽어보시는 것을 추천합니다. 각 단계별로 어떤 정규 표현식을 사용하고, 어떤 알고리즘(예: 델리미터 스택)을 적용하는지 기술적인 세부 사항을 다룹니다.

## 프로젝트 정보

- **런타임**: Bun (최적화된 성능 제공)
- **언어**: TypeScript (엄격한 타입 체크 적용)
- **사양**: CommonMark 0.31.2 (표준 준수)
- **출력 형식**: MDAST (Markdown Abstract Syntax Tree)
- **의존성**: 외부 런타임 의존성 없음 (Zero Dependencies)

## 문서의 목적

이 문서는 단순히 라이브러리 사용법을 알려주는 것을 넘어, 마크다운 파서가 내부적으로 어떻게 동작하는지 교육적인 관점에서 설명하기 위해 작성되었습니다. 개발자가 파서의 동작 원리를 깊이 이해함으로써 프로젝트에 기여하거나, 자신의 프로젝트에서 라이브러리를 더욱 효과적으로 활용할 수 있도록 돕는 것이 목표입니다.

모든 문서는 한국어로 작성되었으며, 기술적인 정확성을 유지하면서도 이해하기 쉬운 용어를 사용하려고 노력했습니다. 문서 내용에 오류가 있거나 개선 제안이 있다면 언제든지 피드백을 환영합니다.

## 문서 기여 방법

문서의 내용을 개선하고 싶으시다면 다음 절차를 따라주세요:

1. 저장소를 포크(Fork)합니다.
2. `docs/` 디렉토리 내의 해당 파일을 수정합니다.
3. 수정 사항이 프로젝트의 코딩 가이드라인(AGENTS.md)을 준수하는지 확인합니다.
4. 변경 사항을 설명하는 풀 리퀘스트(Pull Request)를 제출합니다.

여러분의 기여는 이 프로젝트를 더욱 가치 있게 만듭니다.
